In-context docs

Once you have saved items, a full-width search row sits above two tall cards: the encrypted list (left) and the reader or editor (right). On small screens those cards stack. Idle lock and timers live under Vault → Session. Ciphertext stays on the server; your vault key exists only in this tab after you unlock.

Unlock to decrypt

Choosing an item loads its envelope into the right card. If this tab is still locked, you will be asked for your vault secret (same flow as Settings → Vault Profile). After unlock, decryption runs in the browser.

How this screen works

Search. Only appears when the vault is not empty. It filters using the label shown on each row (from item type), the raw type string, server id, and optional searchable keywords you saved on create or edit. Type #12 style tokens to match a display number exactly.

List. The left card scrolls on its own. Items show as tiles in two columns when the column is wide enough. The lock icon reflects whether this tab currently holds your vault key.

Detail. Select a row to load it on the right. The header shows category, label, and server id. Searchable words is a plaintext hint stored for search only; it is not part of your encrypted JSON. Decrypted contents lists fields from the ciphertext after unlock. Use Edit when it is offered to change values inline; copy buttons sit beside each field.

Practical tips

Refresh. The list loads when you open this page. Reload the browser or navigate away and back if you just created items elsewhere and they are missing.

Edit. After decrypting, choose Edit in the right card. Save returns you to read view with an updated list row; Cancel drops unsaved changes.

Delete. While editing, use the red trash control next to Cancel to remove the server record permanently.

At a glance

Plaintext fields appear only in the right card after a successful decrypt in this tab.
List and search use server metadata; ciphertext bodies download only for the item you select.
