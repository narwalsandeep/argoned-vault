# Central CSS

All app CSS lives here. Single entry: **index.css**.

| File | Purpose |
|------|--------|
| **index.css** | Entry: imports Tailwind, theme, layout. Do not reorder. |
| **theme.css** | Design tokens only (`@theme`). Colors and tokens used via Tailwind (e.g. `bg-app-bg`, `bg-app-elevated` for secondary buttons). |
| **layout.css** | Shared layout/utility classes (e.g. `.page-content`, `.btn-primary` / `.btn-secondary`, `.control-btn-*`). Content width lives here. |

- **Auth & legal (public):** `.auth-shell`, `.auth-shell-body` (wraps the centered card), `.auth-legal-footer` (Terms/Privacy, bottom-right of viewport), `.auth-split`, `.auth-panel-left` / `.auth-panel-right`, `.auth-form-header`, `.auth-field-label`, `.auth-input`, `.auth-checkbox-row`, `.auth-links`. Legal/static copy: `.auth-shell--legal`, `.auth-legal-wrap`, `.auth-legal-brand`, `.auth-legal-card`, `.auth-legal-section`, `.auth-legal-foot` (with `.docs-prose` for body text).
- **Content width:** `.page-content` uses `w-full max-w-none items-stretch` so shell pages span the main column. **Settings**, **Account**, **Vault** (`/vault/items`), and **Dashboard** use `.settings-layout` (`.settings-main` ~9/12 + `.settings-aside` ~3/12 for help / shortcuts). On `lg+`, `.settings-aside` has **no vertical rule** (no extra left padding; column spacing is the layout grid gap only). **Alert** and **Status** use `page-pad--settings settings-page-fill` + `.app-page-layout` > `.app-page-main` (~9/12, no aside). **Docs** and **Pricing** use `.app-page-main.app-page-main--full` for **12/12** width under the shell. Auth/marketing screens use their own `max-w-*` inside `auth-*` classes.
- **Pricing:** `.pricing-tier-grid`, `.pricing-tier-card` / `.pricing-tier-card--featured`, `.pricing-tier-badge` / `.pricing-tier-badge--accent`, `.pricing-tier-*` text utilities, `.pricing-security-grid` & `.pricing-security-item*`. Repo URL: `features/pricing/pricing.constants.ts`.
- **Dashboard / Docs metrics:** `.dashboard-stat-grid`, `.dashboard-stat-card` (CSS grid: icon + body on row 1, `.dashboard-stat-detail` as a **direct child** of the card spans full width on row 2). Optional `.dashboard-stat-card--live`. Icon-less cards omit `.dashboard-stat-icon` (single-column grid). **Docs page:** `.docs-hero*`, `.docs-page-grid`, `.docs-toc*`, `.docs-main-column`, `.docs-pillar*`, `.docs-callout`, `.docs-def-*`, `.docs-section-title` / `.docs-section-kicker`, `.docs-step-list`, `.docs-shell-nav*`, `.docs-body`, `.docs-section`, `.docs-crypto-ref-list`, `.docs-api-list`, `.docs-route-group*`. **Docs prose & tables:** `.docs-prose`, `.docs-route-table` (legacy). **Shell right rail:** `.app-rail-end-stack`.
- **Create (+) flow:** `.settings-layout` + `.settings-aside` help rail on **Create** (`/new`), **Credentials** chooser + **credential form** (`/new/credentials/:subtype`), and **simple vault item** forms (`/new/item/:kind` for id, password, key, secure-note), same ~75% / ~25% as Account/Vault. **Intro:** `.create-flow-intro` (full-width, small muted body). **Choice cards:** `.create-flow-back-row`, `.create-choice-grid` (1 column mobile, 2 from `sm`, equal row heights), `.create-choice-card` (fills grid cell; descriptions wrap, row height follows the tallest card) / `.create-choice-card--soon`, `.create-choice-card__icon`, `.create-choice-card__body`, `.create-choice-card__title-row`, `.create-choice-card__title`, `.create-choice-card__desc`, `.create-choice-card__badge` / `.create-choice-card__badge--soon` (right-aligned in title row). Credential **form** uses `.settings-section-card` + `.tab-panel-heading` + `.control-form-grid` / `.control-split` (single column, like Change password). Schemas live in `credential-form-schema.ts` (Website, Generic, API key, App, Database, SSH, Email, Software license). Copy for each option lives on `CreateableItemOption.description` and `CredentialTypeOption.description`.
- No component-level CSS files; use Tailwind classes in templates.
- **In-app forms (any shell page):** use `.control-field`, `.control-form-grid`, `.control-actions-row` (right-aligned), `.control-actions-group` (segmented button group: shared shadow, `-space-x-px` merged borders, direct `button.control-btn-*` children only), `.control-btn-primary`, `.control-btn-secondary`, `.control-btn-danger` in `layout.css` (optional `.control-tab` / `.control-tab--active` for tab strips). **Settings fields:** `.control-split` + left `.control-split-label` + right `.control-split-input` (or `<select class="control-split-input">`), one bordered row, light label pane (`bg-app-elevated`).
- **Tab / card headings:** `.tab-panel-heading` + `.tab-panel-heading-title` • `.tab-panel-heading-kicker` (see Settings tab panels).
- **Settings help rail (right column):** `.settings-help-*`, flat layout (hero = bottom border only; callout = left accent bar; tips = divided rows; highlights = no mini-cards). Shortcut links: `.settings-help-nav-links`, `.settings-help-nav-link` (+ title/text). **Section cards:** `.settings-section-card` uses a lighter border/background (still one container per form area).
- **Vault unlock modal:** `.vault-unlock-overlay`, `.vault-unlock-doors`, `.vault-unlock-modal-shell` / `rim` / `inner`, compact secure dialog with sliding “vault door” + panel entrance (`layout.css`; `prefers-reduced-motion` disables motion). Secret field: `.vault-unlock-secret-input` (centered; typed password uses brand red via `color` / `-webkit-text-fill-color`, placeholder muted).
- Add new global classes in **layout.css** (or a new file and import in index.css).
- Add new tokens in **theme.css**.
